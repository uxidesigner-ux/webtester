JSON 기반 리포트 생성 플랫폼

이 프로젝트는 관리자 도구(/admin/)로 초안을 만들고, content/reports/*.json를 통해 정적 리포트(/reports/)를 발행하는 GitHub Pages 중심 플랫폼입니다.

리포트 보기 관리자 열기

핵심 경로 안내

/admin/

초안 작성/편집 도구입니다. localStorage 저장, JSON import/export, preview를 제공합니다.

/reports/

build 결과로 생성된 발행 리포트 목록입니다. 개별 리포트는 /reports/{slug}/에서 확인합니다.

운영 방식 (Draft vs Published)

Draft

  • 저장 위치: 브라우저 localStorage
  • 관리 경로: /admin/
  • 배포 반영: 자동 반영되지 않음

Published

  • 원본 위치: content/reports/*.json
  • 생성 경로: dist/reports/* (build 결과)
  • 배포 반영: commit + build/deploy 필요

관리자 사용 흐름

  1. 관리자(/admin/)에서 리포트 작성 및 preview 확인
  2. JSON 내보내기로 발행용 파일 생성
  3. content/reports/{slug}.json 저장/커밋
  4. npm run build로 정적 리포트 생성
  5. GitHub Pages 배포 후 /reports/에 반영 확인

프로젝트 제약

  • DB/CMS/백엔드 없이 정적 자산 기반으로 운영
  • 런타임 GitHub write 없음 (관리자 페이지는 저장 도구만 제공)
  • GitHub Pages 하위 경로(/webtester/)를 고려해 상대 경로 사용
